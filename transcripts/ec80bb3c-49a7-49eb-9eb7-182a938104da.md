## Option 1: Media Processing App
**Core idea:** Upload images/videos, queue async jobs for resizing, format conversion, thumbnail generation, watermarking.

**Queue demonstration:**
- Chained jobs (upload → resize → thumbnail → notify)
- Job batching (process multiple images at once)
- Failed job handling (corrupted files)

**Complexity:** Medium-High
- Requires `intervention/image` or FFmpeg integration
- Storage concerns (local vs S3)
- Binary processing = harder to debug and test

**Time estimate:** 6–10 hours for a meaningful demo

**Strengths:**
- Visually compelling output
- Naturally maps to async (processing takes real time)
- Covers chaining, batching, retries well

**Weaknesses:**
- FFmpeg dependency is environment-sensitive
- Heavy on infrastructure setup, lighter on pure Laravel queue concepts

---

## Option 2: CSV/Data Import Pipeline
**Core idea:** Upload a large CSV, queue rows in batches for validation, transformation, and DB insertion, with progress tracking and error reporting.

**Queue demonstration:**
- Job chunking (split CSV into batches of N rows)
- Progress tracking via DB or cache
- Failed rows collected and reported back
- Notifications when import completes

**Complexity:** Low-Medium
- Pure PHP/Laravel — no binary dependencies
- Easier to fake scale (generate 10k row CSVs trivially)
- Straightforward to unit test

**Time estimate:** 4–7 hours

**Strengths:**
- Clean separation of concerns
- Progress feedback is a natural, impressive feature
- Easy to demonstrate failure/retry semantics on bad data
- Highly relevant to real-world use cases

**Weaknesses:**
- Output is less visual (rows in a table vs rendered images)
- Might feel "ordinary" compared to media

---

## Option 3: Report Generation & Delivery Pipeline
**Core idea:** User requests a report (e.g., sales summary, user activity), system queues PDF generation, then emails it when ready.

**Queue demonstration:**
- Queued Mailables (Laravel's built-in queue integration)
- Delayed jobs (schedule delivery in X minutes)
- Job middleware (throttling, uniqueness)

**Complexity:** Low
- Laravel has native queued mail and PDF packages (`barryvdh/laravel-dompdf`)
- Mostly gluing built-in Laravel features together

**Time estimate:** 3–5 hours

**Strengths:**
- Fastest to complete
- Demonstrates queued notifications/mail natively
- Good for showing `delay()`, `onQueue()`, job middleware

**Weaknesses:**
- Least technically interesting — feels like a tutorial demo
- Limited scope for batching, chaining, or complex job orchestration
- Hard to impress a technical evaluator

Media processing is richer but you can lose hours to environment issues. Report generation is too thin technically.

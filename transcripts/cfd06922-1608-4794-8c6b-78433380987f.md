Here's a practical guide to using Claude Code efficiently:

---

## Essential Commands

| Command | Purpose |
|---------|---------|
| `claude` | Start interactive session |
| `claude "task"` | One-off task, then exit |
| `claude -p "query"` | Quick query, exit immediately |
| `claude -c` | Continue most recent session |
| `claude --resume name` | Resume a named session |
| `claude -n "name"` | Start a named session |

---

## Key Keyboard Shortcuts

| Shortcut | What it does |
|----------|-------------|
| `Shift+Tab` | Toggle permission modes (Normal → Plan → Auto-Accept) |
| `Esc` | Stop Claude mid-action |
| `Esc` + `Esc` | Open rewind menu (undo changes) |
| `Ctrl+G` | Edit your prompt in a full editor |
| `Ctrl+O` | Toggle verbose output |
| `Ctrl+B` | Background a running task |
| `!` prefix | Run bash directly: `! npm test` |
| `@` prefix | Reference a file: `@src/auth.js` |
| `?` | Show all keyboard shortcuts |

---

## Permission Modes (Shift+Tab to cycle)

- **Normal** — prompts you to confirm edits/commands (default)
- **Plan** — read-only, Claude explores but makes no changes (great for understanding code)
- **Auto-Accept** — auto-approves file edits (use when you trust the direction)

**Workflow**: Plan Mode to explore → Normal Mode to implement

---

## Slash Commands

| Command | Purpose |
|---------|---------|
| `/help` | See all commands |
| `/clear` | Reset context (start fresh) |
| `/compact` | Summarize old context to free space |
| `/init` | Generate a starter `CLAUDE.md` for your project |
| `/memory` | View/edit Claude's persistent memory |
| `/hooks` | Browse configured hooks |
| `/rename` | Name the current session |

---

## CLAUDE.md — Project Instructions

Run `/init` to generate one. Claude reads this at the start of every session. Keep it under 200 lines and include:
- Build/test commands (`npm test`, `php artisan test`)
- Code style rules specific to your project
- Architecture decisions

Your project already has transcript hooks configured — you can document that in `CLAUDE.md` too.

---

## Most Effective Workflow

```
1. Enter Plan Mode (Shift+Tab) → ask broad questions
2. Get Claude to outline what needs to change
3. Switch back to Normal Mode → implement
4. Run tests: ! php artisan test
5. /clear between unrelated tasks
```

---

## Pro Tips

- **Name sessions**: `/rename auth-feature` so you can resume them later
- **Undo anything**: `Esc + Esc` opens a rewind menu to restore code/conversation
- **Reference files directly**: `explain @app/Http/Controllers/AuthController.php`
- **Clear context often**: Long sessions degrade quality — `/clear` between tasks
- **Give verification criteria**: "Write a test first, then implement" — Claude checks its own work
